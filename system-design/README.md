## 🎯 学习路线

```
1. 系统设计基础（需求分析、容量估算）
   ↓
2. 高并发架构（负载均衡、CDN、缓存）
   ↓
3. 高可用架构（熔断、降级、限流）
   ↓
4. 微服务拆分（DDD、服务边界）
   ↓
5. 数据一致性（最终一致性、分布式事务）
   ↓
6. 实战案例（短链系统、秒杀系统、Feed 流）
```

---

## 💡 学习建议

- **从需求出发**：先问清楚 QPS、延迟、一致性要求
- **分层设计**：接入层→服务层→数据层
- **权衡取舍**：没有完美方案，只有最适合的
- **多画架构图**：用图表表达设计思路
