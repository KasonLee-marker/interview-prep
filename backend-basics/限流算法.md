## 📚 核心算法

### 1. 令牌桶算法（Token Bucket）

#### 原理
系统以固定速率往桶里放令牌，请求来了先拿令牌，拿到才能处理，拿不到就被限流。

#### 特点
- ✅ 允许突发流量（桶里有存量令牌）
- ✅ 平滑限流
- ⚠️ 分布式场景需保证原子性

#### Redis 实现（Lua 脚本）

```lua
-- KEYS[1]: 令牌桶 key
-- ARGV[1]: 当前时间戳 (毫秒)
-- ARGV[2]: 桶容量
-- ARGV[3]: 令牌补充速率 (个/秒)
-- ARGV[4]: 请求需要的令牌数

local key = KEYS[1]
local now = tonumber(ARGV[1])
local capacity = tonumber(ARGV[2])
local rate = tonumber(ARGV[3])
local needed = tonumber(ARGV[4])

-- 获取当前令牌数和时间戳
local bucket = redis.call('HMGET', key, 'tokens', 'last_time')
local tokens = tonumber(bucket[1]) or capacity
local last_time = tonumber(bucket[2]) or now

-- 计算新增令牌数（惰性计算）
local elapsed = (now - last_time) / 1000.0
local new_tokens = elapsed * rate
tokens = math.min(capacity, tokens + new_tokens)

-- 尝试获取令牌
if tokens >= needed then
    tokens = tokens - needed
    redis.call('HMSET', key, 'tokens', tokens, 'last_time', now)
    redis.call('EXPIRE', key, 3600)
    return 1  -- 成功
else
    redis.call('HMSET', key, 'tokens', tokens, 'last_time', now)
    redis.call('EXPIRE', key, 3600)
    return 0  -- 失败
end
```

#### Java 调用示例

```java
public class TokenBucketRateLimiter {
    private RedisTemplate<String, String> redisTemplate;
    private String script;
    
    public boolean tryAcquire(String key, int permits) {
        Long result = redisTemplate.execute(
            new DefaultRedisScript<>(script, Long.class),
            Collections.singletonList(key),
            String.valueOf(System.currentTimeMillis()),
            "100",    // 容量
            "10",     // 速率：10 个/秒
            String.valueOf(permits)
        );
        return result == 1;
    }
}
```

### 2. 漏桶算法（Leaky Bucket）

#### 原理
请求像水一样流入桶，桶以固定速率漏水（处理请求）。桶满了就溢出（拒绝请求）。

#### 特点
- ✅ 强制恒定速率输出
- ✅ 平滑流量，保护下游系统
- ❌ 无法应对突发流量

#### Redis 实现

```lua
-- KEYS[1]: 漏桶 key
-- ARGV[1]: 当前时间戳
-- ARGV[2]: 桶容量
-- ARGV[3]: 漏水速率 (个/秒)

local key = KEYS[1]
local now = tonumber(ARGV[1])
local capacity = tonumber(ARGV[2])
local rate = tonumber(ARGV[3])

-- 获取当前水量和最后更新时间
local bucket = redis.call('HMGET', key, 'water', 'last_time')
local water = tonumber(bucket[1]) or 0
local last_time = tonumber(bucket[2]) or now

-- 计算漏掉的水量
local elapsed = (now - last_time) / 1000.0
local leaked = elapsed * rate
water = math.max(0, water - leaked)

-- 尝试加水
if water + 1 <= capacity then
    water = water + 1
    redis.call('HMSET', key, 'water', water, 'last_time', now)
    redis.call('EXPIRE', key, 3600)
    return 1  -- 成功
else
    return 0  -- 溢出，拒绝
end
```

### 3. 滑动窗口算法（Sliding Window）

#### 原理
把时间分成小格子，统计窗口内的请求数。

#### 特点
- ✅ 精确控制时间窗口内的请求数
- ✅ 平滑过渡，避免临界问题
- ⚠️ 内存占用较高

#### Redis ZSET 实现

```lua
-- KEYS[1]: 滑动窗口 key
-- ARGV[1]: 当前时间戳 (毫秒)
-- ARGV[2]: 窗口大小 (毫秒)
-- ARGV[3]: 窗口限制数

local key = KEYS[1]
local now = tonumber(ARGV[1])
local window_size = tonumber(ARGV[2])
local limit = tonumber(ARGV[3])

-- 删除窗口外的数据
local window_start = now - window_size
redis.call('ZREMRANGEBYSCORE', key, '-inf', window_start)

-- 统计当前窗口内的请求数
local count = redis.call('ZCARD', key)

-- 判断是否超限
if count < limit then
    -- 添加当前请求
    redis.call('ZADD', key, now, now .. ':' .. math.random())
    redis.call('EXPIRE', key, window_size / 1000 + 1)
    return 1  -- 成功
else
    return 0  -- 失败
end
```

## 📊 算法对比

| 对比维度 | 令牌桶 | 漏桶 | 滑动窗口 |
|---------|--------|------|---------|
| **限流粒度** | 瞬时速率 | 固定速率 | 时间窗口总量 |
| **突发流量** | 允许（桶容量内） | 不允许 | 允许（窗口内） |
| **实现复杂度** | 低 | 低 | 中 |
| **内存占用** | 低 | 低 | 高（ZSET） |
| **Redis 数据结构** | Hash | Hash | ZSET |
| **适用场景** | API 限流、用户限流 | 消息队列、写数据库 | 精确限流、防刷 |

## 💡 核心问题解答

### Q1: 为什么不需要定时任务补充令牌？

**答案**：使用**惰性计算**思想。

```
不是"每秒添加令牌"，而是"每次请求时计算应该添加了多少令牌"

时刻 T0: 拿走 1 个，剩 99，记录时间
时刻 T15: 过了 15 秒，补 150 个→上限 100，拿走 1 个

结果：和每秒真的加令牌完全一样！
```

**好处**：
- ✅ 不需要后台定时任务
- ✅ 没请求时不消耗资源
- ✅ 数学上完全等价

### Q2: 如何选择 Redis 数据结构？

| 算法 | 推荐结构 | 原因 |
|------|---------|------|
| 令牌桶 | Hash | 存储 tokens 和 last_time 两个字段 |
| 漏桶 | Hash | 存储 water 和 last_time 两个字段 |
| 滑动窗口 | ZSET | 需要按时间排序 + 范围删除 |

### Q3: 如何保证分布式原子性？

**答案**：使用 **Redis + Lua 脚本**

- Lua 脚本在 Redis 中是原子执行的
- 多个命令打包成一个脚本
- 避免并发问题

```

实现：用户 ID 作为 key，令牌桶限流
参数：容量=10，速率=10 个/秒
```

### 3. 防刷、防爬虫

```
场景：防止恶意刷接口
实现：滑动窗口 + IP 维度限流
参数：1 分钟内最多 100 次请求
```

### 4. 保护下游系统

```
场景：数据库最多承受 1000 QPS
实现：漏桶算法，强制匀速
参数：漏速=1000 个/秒
```
