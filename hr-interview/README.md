## 🎯 准备要点

### 1. 自我介绍（1-3 分钟）

```
模板：
- 基本信息（姓名、学校、专业）
- 技术栈（擅长什么）
- 项目经历（1-2 个亮点项目）
- 为什么应聘这个岗位
```

### 2. 项目经历（STAR 法则）

```
Situation（情境）：项目背景
Task（任务）：你的职责
Action（行动）：你做了什么
Result（结果）：量化成果
```

### 3. 离职原因

```
✅ 好的回答：
- 寻求更好的发展机会
- 技术栈不匹配
- 业务方向调整

❌ 避免：
- 抱怨前公司/领导
- 薪资太低（可以作为因素之一，但不要是唯一原因）
```

### 4. 优缺点

```
优点：
- 结合岗位需求
- 有具体例子支撑

缺点：
- 真实的但不是致命的
- 正在改进中
```

## 💡 回答技巧

### STAR 法则详解

```
Situation（情境）:
"我在上一家公司负责一个日活 100 万的电商平台..."

Task（任务）:
"我的任务是优化下单接口的响应时间，从 500ms 降到 200ms..."

Action（行动）:
"我做了三件事：1) 引入 Redis 缓存热点数据 2) 优化 SQL 索引 3) 异步化非核心逻辑..."

Result（结果）:
"最终响应时间降到 150ms，提升了 70%，用户转化率提升了 5%"
```

### 薪资谈判

```
- 先了解市场行情
- 给出区间而不是具体数字
- 强调自己的价值
- 综合考虑薪资、发展、团队等因素
```

## 📝 常见问题清单

### 自我介绍类
- [ ] 请做个自我介绍
- [ ] 介绍一下你最有成就感的项目
- [ ] 说说你的技术成长路径

### 项目经历类
- [ ] 项目中遇到的最大挑战是什么？如何解决的？
- [ ] 你在项目中的角色是什么？
- [ ] 如果重来一次，你会怎么改进这个项目？

### 职业规划类
- [ ] 未来 3-5 年的职业规划是什么？
- [ ] 为什么选择我们公司？
- [ ] 你想在团队中扮演什么角色？

### 情景问题类
- [ ] 如果和产品经理意见不合怎么办？
- [ ] 线上出现紧急 bug 怎么处理？
- [ ] 如何平衡技术债务和业务需求？

## 💪 软技能考察

| 能力 | 考察方式 | 准备要点 |
|------|---------|---------|
| 沟通能力 | 情景问题 | 清晰表达、倾听理解 |
| 团队协作 | 项目经历 | 合作案例、冲突解决 |
| 学习能力 | 技术成长 | 自学经历、新技术掌握 |
| 抗压能力 | 情景问题 | 高压下的表现 |
| 问题解决 | 项目挑战 | 分析问题、解决问题的思路 |
